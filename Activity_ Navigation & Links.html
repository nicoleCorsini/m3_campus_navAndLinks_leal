Activity: Navigation & Links

  • Due Sunday by 11:59pm
  • Points 5
  • Submitting a website url
  • Available until Jan 28 at 11:59pm

 

Activity Objectives:

By the end of this lesson, you will be able to:

  • Write code using a relative link.
  • Write code using an absolute link.
  • Write code using a fragment link.
  • Write code using an email link.
  • Write code using a telephone number link.
  • Write code for navigation using the nav, ul, li, and a elements.
  • Create three html pages that correctly link through the use of navigation.

 

Instructions:

As students in this course and future web developers, you will be asked to create pages for the web that include links to other resources on the web or elsewhere, including: other pages within your site, sections/topics on your site, email addresses, telephone numbers, and outside links. Today we will practice implementing one of the three things we use to define what the web is: "Text files, known as HTML files, containing links to other text files."

 

Part One:

We will be working in Visual Studio Code today, using GitHub and GitHub Desktop. Before we start, make sure that you have Visual Studio Code open on your computer (or a classroom computer) and signed in to settings sync. Also ensure that you have GitHub Desktop open and are logged in to your GitHub account there and in the browser. Once that is done, you can complete this step.

    1. Make sure that you are signed in to your GitHub account, and go to the repository (m3_campus_navAndLinks) in your browser.
    2. Fork a copy of the repository into your account for personal use, not to contribute (instructions can be found in the document, follow the link to find them).
    3. Clone your new/forked repository to your computer using GitHub Desktop. Be sure to clone this into your m3 folder when choosing the path.

NOTE:

On your version control-based assignments and activities it will be important to save this in your repository. We name a lot of files index.html, so in order to avoid confusion it's important that you stay organized. index.html means "home page" in computer language.

 

Part Two:

We have our repository/files on our computer, and in order to add and edit files in that repository we will use Visual Studio Code.

    1. From GitHub Desktop, open the repository in Visual Studio Code (Control/Command + Shift + A), then open the index.html file in the repository.
    2. Review the content and code in this file, and notice that comments have been added to help us see where we have work to complete.
    3. We will start by completing the code in this file, then we will complete the contact.html file, then we'll be creating a new web page for this site. All of the pages will be linked to each other by us during this activity.
    4. If done correctly, the index page should look like this Solution Image Download Solution Image (you can look back at this after completing the activity).

 

Part Three:

We've looked over our files, now let's finish the code. First, we'll work on the index.html file. Remember, the landing/home page for your website should always be named "index.html" in all lowercase letters. This is the filename that a browser will look for in your project and will open it automatically.

    1. In the head of the index.html file, add your name to the meta author tag, then save the file.
    2. Following along with the video, mark up the index.html page with the appropriate links. When we've finished, the index.html page should have:
      1. Relative Links: 2
      2. Absolute Links: 1
      3. Dummy Links: 3 (two are coded for you)
      4. Email Links: 2
      5. Telephone Links: 1
      6. Fragment Links: 7
      7. Relative/Fragment Links: 1
    3. Save the index.html file.
    4. Validate the index.html Links to an external site. file and correct any errors.
    5. Compare your document outline to the one displayed below and ensure that yours matches (you can do this from the validator, just check the box next to the text "outline" and run the validator again). This will help you to see whether your content is nested and tagged properly.
    6. Save the index.html file again after errors have been corrected and your file is valid, then commit your changes to your repository and push them to GitHub. Remember, a message is required with every commit.

 

Part Four:

We've finished with the index.html file, now let's complete the contact.html file for our three page website.

    1. Be sure you saved, committed, and pushed the changes related to your index.html file.
    2. Open the contact.html file from the repository in Visual Studio Code.
    3. Review the content and code in this file, and notice that comments have been added to help us see where we have work to complete.
    4. Update the h1 on the page to say "Navigation and Links | Contact". (Not sure how to code the vertical line? It's called a "pipe" and should be on your keyboard below the delete key/above the enter/return key. It usually shares the same key as the backslash.)
    5. Some of that work is the same that we completed in parts of the index.html file, so you can compare to that file as we work to complete the code on this page together.
    6. When the page is fully marked up, you should have the following links:
      1. Relative Links: 2
      2. Dummy Links: 1
      3. Telephone Links: 1
      4. Email Links:  1
      5. Relative/Fragment Links: 1
    7. Save the contact.html file.
    8. Validate your file Links to an external site. and correct any errors. (You are welcome to look at the outline for this page in the validator as well).
    9. Save the contact.html file again after errors have been corrected and your file is valid, then commit your changes to your repository and push them to GitHub. Remember, a message is required with every commit.

 

Part Five:

Two of our three pages are complete now, but for the last one we need to create a new file and complete the code.

    1. Be sure you saved, committed, and pushed the changes related to your index.html and contact.html files.
    2. Create a new file (Control/Command + N) in Visual Studio Code and save it as (Control/Command + Shift + S) about.html inside of your repository folder for this activity.
    3. Copy all the code from your index.html file and paste it into your about.html file.
    4. Confirm you are editing the about.html file and delete all the content after the header and before the closing body tag. Be sure your header and footer (with all of their content) are still intact.
    5. Change the title tag and h1 tag to read "Navigation and Links | About". (Not sure how to code the vertical line? It's called a "pipe" and should be on your keyboard below the delete key/above the enter/return key. It usually shares the same key as the backslash.)
    6. Below the header add a paragraph that says: "This is the about page." (Note: we are not concerned that this page does not have appropriate content. This part of the assignment is to ensure your navigation works correctly).
    7. When the page is fully marked up, you should have the following links:
      1. Relative Links: 2
      2. Dummy Links: 1
      3. Telephone Links: 1
      4. Email Links:  1
      5. Relative/Fragment Links: 1
    8. Validate the file Links to an external site. and correct any errors.
    9. Save the about.html file, commit your changes to your repository, then push those changes to GitHub. Remember, a message is required with every commit.

 

Part Six:

Now we have a three page website that includes navigation and links! Let's publish that to the web using GitHub Pages so we can make sure that our links work in the browser.

    1. From your repository on GitHub, click on the "Settings" link near the upper-right corner of the page .
    2. Under “Code and automation” click on the “Pages” menu item.
    3. Under "Build and deployment" and "Source", select "Deploy from a branch".
    4. Under "Build and deployment" and "Branch", use the Branch drop-down menu and select a publishing source (main).
    5. Use the drop-down menu to select a folder for your publishing source (root).
    6. Click Save, and allow GitHub a minute to publish. You will see a link appear on the page with a refresh.
    7. Click on the link to view your published webpage! (Note that the URL to your hosted site in GitHub Pages is not the same as your repository’s URL).

 

Part Seven:

View your published site in the browser and confirm all the navigation and other links work on each page.

NOTE:

In order for relative links to work correctly, all three html pages MUST reside in the same folder/location. If you followed the directions in the video and above, you should be fine, but remember on future projects that file organization is important, which is why we started the course by setting up folders for our work in the class!

If any of your links do not work as expected, go back and fix the errors, commit and push the changes to GitHub, then test again in the browser after the site updates and you refresh the page.
